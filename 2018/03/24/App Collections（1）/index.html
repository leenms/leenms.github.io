我眼中的少数派

2018-03-24

关于少数派

少数派 是一个数字生活内容社区，所有能够提升工作、学习效率，以及生活幸福感的数字化产品、内容，都可能在某一天你习惯性地浏览首页时，出现在眼前。

这些内容完全由平台上的作者产生，这些作者们被称为「Matrix 作者」。Matrix，也就是少数派的写作平台，用户首先需要申请成为这个社区的作者，才可以发表内容。我简单地把 Matrix 理解成是少数派的文章库。值得一提的是，Matrix 也是《黑客帝国》里那个连结了虚拟空间与现实世界，并孕育着人类以及人类意识的庞大系统。

「少数」在哪

定位精准且质量优秀的内容是少数派的核心竞争力，就不再赘述了，长期阅读下来的人应该可以体会，强烈建议你将这个平台纳入你获取技巧与信息的「底层系统」中来。这篇文章重点分析一下对于一个内容平台的体验要素来说，我认为少数派做得最不一样的地方。

内容流通漏斗模型

内容从生产出来的一刻到真正为用户所取，以最核心的首页场景为例，中间流通的过程可以看成一个漏斗模型（也可称为倒金字塔模型），并且根据内容所处阶段的不同，可以将这个漏斗分为四个部分。

那么决定一个内容平台的体验要素的，到底是这个漏斗中的哪些元素呢？

那些产品追求的

在这个漏斗中，很多产品追求的是一个绝对值，也就是用户最后能取之所用的内容的绝对多少（C4），甚至到了 C3 就满足了。现在很多平台都在不遗余力地于生产以及向用户传递大量的内容，其中还有很多同质化的。

其实追求 C4 的最大化，对用户体验来说并非一定有问题，如果 C4 足够接近 C3，其实就是行得通的路，因为这意味着触达用户的内容中，绝大部分都是用户想看的、需要的。典型的例子就是今日头条，它会有无数的内容让你刷，让你停不下来，而且不容易产生抗拒。两点因素，一是头条提供的内容都不会过于深度，而是以快餐内容为主；另一点就是得益于强大的技术能力，能够使 C4/C3 足够趋近于 1，推荐算法解决的就是这个问题，它在尽可能提供你本来就感兴趣的内容。其他头条系产品同理。

传统的内容及资讯产品，是一家百货超市，你需要什么你得自己去找；优秀的内容产品是一家提供了外送服务的垂直类生鲜店，你什么时候想吃了就在家掏出手机随便点几下就可以了；今日头条这种，是让你常驻在了大型商场里面，还允许你想住哪一层就住哪一层，因为你可能是喜欢逛服装店的，喜欢逛电玩城的，或者喜欢逛日用超市的。如果某一天你突然说超市逛够了、逛厌了，喜欢上了 3C 产品，今日头条说：「那行你去吧，你先自己多逛几次，我马上去办手续让你从负一层搬到楼上去。」就是这么有求必应。

少数派追求的

一个内容平台的体验要素，确实可以从很多点去考虑。这个时代由于信息的过度产生与流动，传统的门户已经无法顺利地帮人们获取他们想要的信息了，今日头条走了一条最粗暴的路，把你想要的一股脑全拿给你看。而少数派作为一个相对垂直的数字生活内容平台，由于该领域高度重合的用户特性，以及为了质量，其内容体量又无法做得太大的特点，注定就是与今日头条不同的物种。

然而对于整个数字生活领域来说，其定义还是比较广的。回到漏斗模型，这四个部分的内容其实是依次往下流通和转化的，我在这里提取 3 个百分比：

  • P1：平台主动呈现在用户首页的内容占平台所有内容总量的比率（C2/C1）。

  • P2：真正可以触达用户的内容占平台主动呈现在用户首页的全部内容的比率（C3/C2）。

  • P3：用户真正需要、接受的优质内容占可以触达用户的全部内容的比率（C4/C3）。

相对于其他内容平台，少数派是怎样通过控制这 3 个百分比来体现其产品价值以及用户体验的呢。

1. 维持较低水平的 P1：提高 C1，维持合理但较小的 C2

提高 C1，也就是提高平台内容总量。这一点很好解释，任何一个内容产品首先都应该有足够多的优质内容。此外，足够大、足够丰富的 C1，也为各种专栏的建立提供了可行性。

维持 C2，也就是维持每天推送至首页的内容量，不要随时随地都在做大量的更新。很长时间以来，少数派首页的日更量都维持在 6 篇左右，且周末的量还会减半。我翻阅了少数派官方 Telegram 频道推送的消息记录（这个频道每天都会给在群里推送当天首页更新的内容），发现至少半年以来都是维持在这个水平。

这一点带来的体验就是，我不需要经常性的想起少数派上是不是又更新了好多优质内容需要看，生怕漏掉什么。而是当我想起来少数派的时候，就去打开看一看，想不起来的时候，一天两天也没事，因为再当我进来的时候，如果我有兴致，我也就可以去把前两天的内容补起来（这里如果还是遇到内容太多的情况，就涉及到另一款「稍后阅读」App 的使用，它帮了我很多，后面有机会再分享）。长此以往，我与少数派达成了一种非常合拍的平衡，时不时就可以在上面获取到有价值的内容。

当我回想其他几十个内容及资讯类产品，其中很多都会让人产生一种「内容这么多，反正看不完」的「习得性」情绪，唯有实在需要主动去检索什么信息时，才会想起来它们可能还在我的手机上占据着宝贵的存储空间。而其他为数不多的把内容做得优质，又比较节制的产品，我就会经常想起来去浏览一次。

2. 尽量提高 P2：在 C2 已定的情况下，通过更加合理的内容呈现形式提高 C3

对于 C2 的处理，前一点已经讲过，那就是在首页维持合适且较低水平的日更量。此时还需要做的是提高 C3，也就是用户能够接收到或者看到的内容量。这里的含义是虽然内容已经推送到用户的首页，但用户能不能真正接收到是另一回事，因为用户经常就只是刷刷页面浏览一下，不会那么仔细地看。

做到这一点的方法是慎重对待首页出现的每一篇内容，因为它们都是编辑部精心挑选的、最优秀的内容了。为它们每一篇提供足够的版面，以保证用户的视线真的可以触及每篇文章，而不是让用户的视线可以在一屏的页面上随意一扫而过。

同时对于 PC 端 website 尤为重要的是，不要因为屏幕大了很多，就可以随意改变用户逐一线性浏览的方式，那样可能得不偿失，需要谨慎取舍。你的确可以通过堆砌的方式让用户看到你有很多（也许很优秀的）内容，但其实都是走马观花，因为用户此时的阅读模式是快速浏览以尽可能快地接收到你呈现给他的信息。在这样的情况下，你的内容不一定更好地被用户接收到了，但却给了用户一些视觉负担。

少数派在移动客户端以及 PC website 上的处理就非常一致且合理，无论是文章标题，还是题图和摘要，都被恰如其分地展示出来了，你的浏览速度不自觉地就会降下来，从而获得「沉浸式」的阅读体验。此外，还有一个最典型的页面就是文章详情页在用户读完该文章之前看不到任何关联阅读的推荐，反观其他平台就能体会到差别所在，生怕更多其他内容你不会去看。

3. 尽一切可能提高 P3：在 C3 已定的情况下，从内容本身来提高 C4

这一部分要做的是确保内容的质量（包括内容本身带来的阅读体验），从而让用户最大化获取价值。

一是少数派 Matrix 社区的运作机制（文章开头已经提及），除了平台官方会邀请各领域的作者贡献内容之外，普通用户想发表内容首先需要申请成为 Matrix 作者，审核通过成为作者之后即可发表作品在 Matrix 社区；二是少数派提倡了一套良好的社区规范以及写作排版规范。你可以感受到少数派上的一切都是围绕「写作」及「阅读」这两个核心来运转的。关联阅读：

  • 《少数派平台规范》

  • 《少数派用户晋升指南》

  • 《少数派写作排版指南》

尾巴

如果一个好的产品是有温度且通人性的，那么对于少数派来说，根据上文分析的 3 个点，总结起来就是以下 3 句自白：

  • 我们平台的优质内容有很多，但我们会挑选最值得你花费时间和精力来阅读的内容推荐给你，尽量为你减少这个时代人们普遍都有的认知焦虑和压力。

  • 我们希望从容又自信地提供每一篇精彩的内容而不是因害怕你过早地离开我们的页面而一次性向你砸去一堆内容，好让你可以轻松又沉浸地在它们当中选择你真正需要的。

  • 最后也是最重要的一点，每一篇你看到的内容，都经过了慎重的选题、严格的编审，以让你获取到真正有价值的「高品质数字消费指南」。
